# Flutter Project Rules

## UI Rules

- Prefer StatelessWidget
- Avoid widget trees deeper than 5 levels
- Prefer InkWell over GestureDetector
- Extract reusable widgets
- Preserve Figma spacing exactly
- Prefer dot shorthands for common Flutter UI types (EdgeInsets, BorderRadius, Radius, etc.).

## Styling

- Use ThemeData
- Avoid hardcoded colors
- Extract design tokens

## Architecture

- Feature-first structure
- Riverpod for state management
- Repository pattern
